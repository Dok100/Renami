# PR Checklist

## Funktion

- Verhalten entspricht der Spezifikation
- Fehlerpfade wurden mitgedacht
- relevante Edge Cases wurden geprueft

## Code

- Aenderung ist lokal begrenzt
- Benennung und Struktur sind konsistent
- unnoetige Komplexitaet wurde vermieden

## Tests

- relevante Checks liefen lokal
- neue Logik ist abgedeckt oder begruendet nicht abgedeckt

## Doku

- geaenderte Annahmen sind dokumentiert
- neue Umgebungsvariablen sind in `.env.example`
