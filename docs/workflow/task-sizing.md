# Task Sizing

Codex arbeitet am besten mit Aufgaben, die in einem Durchlauf abgeschlossen oder belastbar angeschnitten werden koennen.

## Gute Task-Groesse

- ein klarer Nutzerwert
- wenige betroffene Module
- klarer Fertig-Zustand
- direkt testbar

## Warnsignale fuer zu grosse Tasks

- mehrere unabhaengige Screens oder APIs gleichzeitig
- neuer Datenentwurf plus UI plus Deployment in einem Schritt
- unklare Abhaengigkeiten
- keine klare Definition von fertig

## Regel

Wenn ein Task mehr als ein testbares Ergebnis erzeugen soll, teile ihn.
