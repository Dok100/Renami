# Codex Loop

Empfohlene Standardschleife fuer wiederkehrende Arbeit mit Codex.

## 1. Verstehen

- betroffene Dateien lesen
- vorhandene Entscheidungen und Randbedingungen erfassen
- Scope auf ein testbares Inkrement reduzieren

## 2. Festlegen

- relevante Feature-Spezifikation anlegen oder schaerfen
- offene Annahmen sichtbar machen
- Erfolgskriterium fuer den Task benennen

## 3. Umsetzen

- zuerst den minimalen korrekten Pfad bauen
- Nebenwirkungen begrenzen
- bestehende Muster im Repository bevorzugen

## 4. Pruefen

- relevante Tests oder Checks ausfuehren
- Grenzfaelle kurz pruefen
- Dokumentation aktualisieren, falls Verhalten geaendert wurde

## 5. Abschliessen

- Ergebnis kurz zusammenfassen
- Restrisiken benennen
- naechsten sinnvollen Schritt nennen

## Effizienz

Wenn Aufgaben unklar oder zu breit werden, nutze `docs/workflow/token-efficiency.md`, um Scope und Kontext wieder zu verkleinern.

Wenn du wiederkehrende Spezialaufgaben erkennst, nutze `docs/workflow/agent-strategy.md`, um bewusst zu entscheiden, ob ein eigener Agent sinnvoll ist.
